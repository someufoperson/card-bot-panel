== ПРИМЕНЕНИЕ ИЗМЕНЕНИЙ ==

1. ДОМЕН В .env
   Открой .env и замени строку CORS_ORIGINS:
   CORS_ORIGINS=["http://localhost:3000","https://твой-домен.com"]

2. ПЕРЕЗАПУСК DOCKER
   docker compose restart nginx backend

3. CLOUDFLARE TUNNEL
   Туннель сейчас указывает на порт 5000 (Flask напрямую).
   Нужно переключить его на порт 80 (Nginx), чтобы роутинг работал правильно.

   Как изменить:
   а) Зайди на dash.cloudflare.com → Zero Trust → Networks → Tunnels
   б) Найди свой туннель → Edit → Public Hostname
   в) Измени Service:
      - Было:  http://localhost:5000
      - Стало: http://localhost:80

   После этого Cloudflare будет слать трафик в Nginx,
   а Nginx сам распределит его между Flask, FastAPI и React.

== ПРОВЕРКА ==

После применения зайди по своему домену и проверь:

  https://твой-домен.com/            → должна открыться React-панель
  https://твой-домен.com/api/v1/     → JSON-ответ от FastAPI (или 404/422 — это нормально)
  https://твой-домен.com/SERIAL      → страница стриминга scrcpy
  DevTools → Network → WS            → SocketIO подключается (статус 101)


== ПРИМЕНЕНИЕ ИЗМЕНЕНИЙ: АВТОРИЗАЦИЯ ПАНЕЛИ ==

1. ЗАДАТЬ ЛОГИН/ПАРОЛЬ В .env
   Открыть .env и заменить значения:

     PANEL_USERNAME=admin
     PANEL_PASSWORD=твой_пароль
     JWT_SECRET=случайная_длинная_строка
     JWT_EXPIRE_DAYS=30

   Сгенерировать JWT_SECRET и сразу записать в .env:
     sed -i "s/JWT_SECRET=.*/JWT_SECRET=$(openssl rand -hex 32)/" .env

   Или вручную: запусти `openssl rand -hex 32`, скопируй результат и вставь в .env.

2. ПЕРЕСОБРАТЬ И ПЕРЕЗАПУСТИТЬ КОНТЕЙНЕРЫ
   (нужно, т.к. добавилась зависимость python-jose и новые файлы)

     docker compose down
     docker compose build --no-cache backend frontend
     docker compose up -d

3. ПРОВЕРИТЬ ЧТО ВСЁ РАБОТАЕТ
   - Открыть панель в браузере — должна появиться форма входа
   - Ввести логин/пароль из .env
   - После входа — панель открывается как обычно
   - Кнопка выхода — иконка внизу левой панели навигации
